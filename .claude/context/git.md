# Git Guidelines

**IMPORTANT**: All git operstions MUST be approved by the user. ALWAYS provide the user with a summary of your intention... and prompt for an approval.

## Commits
An individual commit should NEVER...
- leave any tests broken/failing
- yield any violatuons from the linter/formatter
- make the build fail

An individual commit should...
- ideally change less than 200 lines of code.
- contain changes to files that are coupled/related.


## Pull Requests
Pull requests should...
- consist of 1-3 commits.
- ideally change less thsn 400 lines of code.
