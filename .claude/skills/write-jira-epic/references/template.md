# Epic Template & Field Guidelines

## Template

```
Title: {title}

## Context
{context}

## Objectives & Milestones
1. {objective 1} (Target: YYYY-MM-DD)
2. {objective 2} (Hard Deadline: YYYY-MM-DD)
...

## Links & Resources
- {link 1}
- {link 2}
...
```

---

## Field Guidelines

### Title

Brief, outcome-focused. Describes the end state, not the activity.

- Good: "Checkout Flow Migrated to Event-Driven Architecture"
- Bad: "Migrate checkout flow" or "Implement event-driven architecture"

### Context

Answer: What is the business goal? Which company initiative does this support? Why now?

- Audience is both engineers and senior leadership — avoid jargon, be direct
- 3-5 sentences max
- Use concrete numbers where available; avoid vague claims

### Objectives & Milestones

- Numbered list, ordered chronologically
- Each milestone should be concrete enough to use as a status-update checkbox
- Attach dates where provided: `Target:` for soft dates, `Hard Deadline:` for fixed ones
- Omit date suffix if no date is known — do not invent one

### Links & Resources

- List all user-provided links
- Label each one clearly (e.g. `[RFC] Payment Retry Design`, `[Slack thread] Discussion on scope`)
- If a link was fetched and its content was used, still include it here
- Omit the section entirely if no links were provided
